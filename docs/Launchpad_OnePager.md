# **MVF Launchpad**

*Ship tools fast. Keep them running.*

# 

# THE PROBLEM: We're all building useful stuff. But what happens when someone goes on holiday?

Vibe coding is great, people are solving real problems fast. But these tools become fragile.

| 🏖️ Holiday Risk Tool breaks, maker's on a beach, nobody can fix it | 🧩 Tech Soup Next.js, Vite, Vue, random VPS... impossible to support | 🔓 Security Blindspots No review, unclear data flows, potential risks | 🔍 Hidden Gems Great tools exist but nobody knows about them OR duplication of effort |
| :---- | :---- | :---- | :---- |

# THE SOLUTION: Make it easy to build right, make it easy to find,  make it easy to support 

| 🧰 Starter Kit Pre-configured dev environment. Auth, database, deployment, just works. | 📚 App Library Central directory of all internal tools. Find what exists. Know who built it. | 🤝 Support People who help, not gatekeep. Office hours, code review, migration help. |
| :---: | :---: | :---: |

# HOW IT WORKS: From side project to properly supported 

| 🛠️ Build Starter Kit or your own | → | 📝 Register 2 mins to add to Library | → | 📈 Grow Tool gets traction | → | ✅ Graduate Proper support coverage |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: |

The shortcut: Build on Starter Kit and graduation is almost automatic, you're already compliant.

# 

# APP LIBRARY TIERS: Clear expectations for makers and users 

| 🟡 COMMUNITY Experimental Use at your own risk. Maker-supported only. Great for early tools. | 🔵 VERIFIED Meets Standards Security reviewed, documented, approved infra. Good for teams. | 🟢 SUPPORTED Production-Ready Formal support, backup owners, SLA. Safe for critical workflows. |
| :---: | :---: | :---: |

# WHAT THIS MEANS FOR YOU 

| 🛠️ If you're building tools ✓ Starter Kit gets you deployed in minutes ✓ No more worrying about auth, hosting ✓ Your tool gets visibility ✓ Help available when you need it | 👤 If you're using tools ✓ Find existing tools first ✓ Know what's safe to rely on ✓ Clear who to contact ✓ No more "ask Sarah" | 👔 If you're leading teams ✓ Visibility into what's being built ✓ Risk managed, innovation enabled ✓ Business continuity sorted ✓ No approval gates |
| :---- | :---- | :---- |

# TO BE CLEAR 

| ❌ This is NOT • A new approval process • IT telling you what you can build • Mandatory anything (yet) • A way to slow you down • Another governance committee | ✓ This IS • Making good practices the easy path • Helping useful tools stay useful • Voluntary, with clear incentives • Support when you want it • A way to find what already exists |
